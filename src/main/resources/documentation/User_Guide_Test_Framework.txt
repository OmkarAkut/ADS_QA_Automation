Webdriver Test Framework (04-05-2021)

WebDriver Test Framework is in src > main > java and src > test > java:
    main: Java classes that contain the main test framework and common components
    test: Java classes that contain test scripts, suites, and helpers for specific functional areas

main directories:
    data: global data sets (expected values)
    global: global framework classes (Setup, Driver, Before/After, Login).  Linked through inheritance.
    helpers: common functional classes and methods (Java, Web, Wait, Go, Do, Get, Assert).  Linked through inheritance.
    maps: page maps (i.e., a single source for element locations).  Need to be instantiated to use in scripts.
    users: login users (test users).  User roles are set within ADS.
    main resources: driver.properties (allows user to set local run configuration)

test:
    maptests:  tests that verify page map elements (element is displayed)
    pagetests: tests that verify important page functionality (implements IPageTests)
    scripts: test scripts organized by functional area within ADS
    smoketests: commonly used smoke test scripts
    suites: all suites that organize test scripts into collections (maptests, smoketests, etc.)
    templates: various script and suite templates (copy and rename to use)

Overview:
1)  Most of the test framework classes are linked through inheritance, so an individual test script should extend one of the
these classes, such as Login, or one of the functional area specific helpers.

2) By default, Junit runs @Test methods in no particular order (since it assumes all tests should be independent).
To specify the test method order, add "@FixMethodOrder(MethodSorters.NAME_ASCENDING)" above the test class name, such as:
    @FixMethodOrder(MethodSorters.NAME_ASCENDING)
    public class MyTestScript {}

3) The main test script template is in the templates package: StaticScriptTemplate.

Recommended Practices
1) Within a test script, each @Test method should ideally contain a single assert statement along with a
method name that is descriptive, meaningful, and readable, so failures can be easily understood from a report.
