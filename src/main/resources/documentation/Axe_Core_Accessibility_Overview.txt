Axe Core Accessibility Overview

Axe is an open source accessibility tool that can be run as a chrome extension or as part of a functional test
framework (the Axe core version).  Axe core was added to the ADS test framework.  This is a brief overview.

	The Axe core functionality is composed of:

	1) The Axe class in main > java > webdriver > utilities.  This contains a single method
		"runAxeAccessibilityTestOfPage" that has three parameters:  the WebDriver object, 
		a logger object, and the JUnit test method name.

		runAxeAccessibilityTestOfPage(WebDriver driver, Logger logger, String methodName)

	2) The "axe.min.js" in main > resources.

	3) The "AdsAccessibilityTestSuite" which contains all the tests of each page as a single class.
		Basically, each test method navigates to the page to test and then calls the 
		runAxeAccessibilityTestOfPage method.
	
	4) The following pom entry:
		<dependency>
      		<groupId>com.deque</groupId>
      		<artifactId>axe-selenium</artifactId>
      		<version>3.0</version>
    	</dependency>


	Additionally, in order to have a readable report of the results, log4j2 is used for reporting.  The components
	of log4j2 are:
	
	1) The "log4j2.properties" file in main > resources.  The report is generated and saved in the location
		specified for the "property.filename" property.

	2) The following pom entries:
		<dependency>
      		<groupId>org.apache.logging.log4j</groupId>
      		<artifactId>log4j-api</artifactId>
      		<version>2.12.1</version>
    	</dependency>
    	<dependency>
      		<groupId>org.apache.logging.log4j</groupId>
      		<artifactId>log4j-core</artifactId>
      		<version>2.12.1</version>
    	</dependency>
